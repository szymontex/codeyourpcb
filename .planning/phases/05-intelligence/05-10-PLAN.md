<task type="checkpoint:human-verify" gate="blocking">
  <name>Verify Autorouting (INT-01)</name>
  <what-built>
FreeRouting integration with DSN export, CLI route command, and SES import.
  </what-built>
  <how-to-verify>
1. Create a test board with 3+ components and 2+ nets
   ```cypcb
   version 1

   board test_routing {
       size 50mm x 30mm
       layers 2
   }

   component R1 resistor "0402" {
       value "10k"
       at 10mm, 15mm
   }

   component R2 resistor "0402" {
       value "10k"
       at 30mm, 15mm
   }

   component C1 capacitor "0402" {
       value "100nF"
       at 20mm, 10mm
   }

   net VCC {
       R1.1
       C1.1
   }

   net GND {
       R1.2
       R2.1
       C1.2
   }

   net SIGNAL {
       R2.2
   }
   ```

2. Run: `cypcb route test.cypcb --freerouting /path/to/freerouting.jar`

3. Verify:
   - Progress output during routing
   - .routes file created
   - No errors

4. Open in viewer, check:
   - Traces visible connecting VCC pins
   - Traces visible connecting GND pins
   - Trace width reasonable (default or specified)
   - Vias visible if layer changes needed

5. Add a locked trace and re-route:
   ```cypcb
   trace VCC {
       from R1.1
       to C1.1
       layer Top
       locked
   }
   ```
   - Verify locked trace preserved after re-routing
  </how-to-verify>
  <resume-signal>Type "routing works" or describe issues</resume-signal>
</task>

<task type="checkpoint:human-verify" gate="blocking">
  <name>Verify LSP Integration (DEV-02)</name>
  <what-built>
LSP server with hover, completion, go-to-definition, and DRC diagnostics.
  </what-built>
  <how-to-verify>
1. Install VS Code extension (or configure generic LSP client)
   - Configure to run: `cypcb-lsp --stdio`
   - Associate with .cypcb files

2. Open a .cypcb file in VS Code

3. Test Hover (Ctrl+hover or mouse hover):
   - Hover over component name (R1) -> shows footprint, value, position
   - Hover over net name (VCC) -> shows connected pins
   - Hover over footprint string ("0402") -> shows footprint details

4. Test Completion:
   - Type `component X resistor "` and trigger completion
     - Should suggest footprint names: 0402, 0603, SOIC-8, etc.
   - Type `net TEST { R1.` and trigger completion
     - Should suggest pin numbers

5. Test Go-to-Definition:
   - Click on "R1" in `R1.1` pin reference
   - Should jump to component R1 definition

6. Test Diagnostics:
   - Create intentional DRC error (components too close)
   - Squiggles should appear under problematic component
   - Hover shows error message
  </how-to-verify>
  <resume-signal>Type "LSP works" or describe issues</resume-signal>
</task>

<task type="checkpoint:human-verify" gate="blocking">
  <name>Verify Trace Rendering and Ratsnest</name>
  <what-built>
Trace rendering with actual width, via rendering, and ratsnest display.
  </what-built>
  <how-to-verify>
1. Open routed board in viewer

2. Trace rendering:
   - Traces show actual width (not thin lines)
   - Top layer traces in red/copper color
   - Bottom layer traces in blue (if any)
   - Trace ends are rounded

3. Via rendering:
   - Vias show as copper-colored circles
   - Drill hole visible in center
   - Vias connect top and bottom traces

4. Ratsnest:
   - Toggle ratsnest layer ON
   - Unrouted connections show as yellow dashed lines
   - Toggle ratsnest layer OFF
   - Dashed lines disappear

5. Create partially routed board:
   - Add 4 pins to a net
   - Route only 2 of them manually
   - Ratsnest shows remaining 2 connections
  </how-to-verify>
  <resume-signal>Type "rendering works" or describe issues</resume-signal>
</task>
