Create LSP crate using tower-lsp-server (per RESEARCH.md):

1. **Cargo.toml**:
   ```toml
   [package]
   name = "cypcb-lsp"
   version = "0.1.0"
   edition = "2021"
   description = "Language Server Protocol implementation for CodeYourPCB"

   [[bin]]
   name = "cypcb-lsp"
   path = "src/main.rs"

   [dependencies]
   cypcb-core = { path = "../cypcb-core" }
   cypcb-parser = { path = "../cypcb-parser" }
   cypcb-world = { path = "../cypcb-world" }
   cypcb-drc = { path = "../cypcb-drc" }
   tower-lsp = "0.20"
   tokio = { version = "1", features = ["full"] }
   serde_json = "1"
   dashmap = "5"
   ```

2. **lib.rs**: Re-exports for library usage

3. **main.rs**:
   - Parse args (--stdio default, --tcp optional)
   - Initialize tokio runtime
   - Create LspService with Backend
   - Serve on stdio (per tower-lsp pattern)

4. **backend.rs** (Backend struct):
   - client: Client handle for notifications
   - documents: DashMap<Url, DocumentState>
   - Implement LanguageServer trait:
     - initialize(): Return capabilities (hover, diagnostics, TextDocumentSyncKind::FULL)
     - initialized(): Log ready
     - shutdown(): Clean up
     - did_open(): Parse document, run diagnostics
     - did_change(): Re-parse, re-run diagnostics
     - did_close(): Remove from documents
     - hover(): Delegate to hover module

5. **document.rs** (DocumentState struct):
   - uri: Url
   - content: String
   - version: i32
   - ast: Option<SourceFile>
   - world: Option<BoardWorld> (lazy, for DRC)
   - parse_errors: Vec<ParseError>

6. Add to workspace in root Cargo.toml

  <action>
Implement hover information per CONTEXT.md (basic by default):

1. **hover_at_position(doc: &DocumentState, position: Position) -> Option<Hover>**:
   - Convert LSP position to byte offset
   - Find AST node at offset
   - Return hover info based on node type

2. **Hover content by node type**:

   a. **Component definition** (e.g., cursor on "R1"):
      ```
      **R1** (resistor)
      Footprint: 0402
      Value: 330
      Position: 10mm, 8mm
      Rotation: 0deg
      ```

   b. **Net definition** (e.g., cursor on "VCC"):
      ```
      **Net: VCC**
      Connected pins:
      - J1.1
      - R1.1
      - C1.1
      Width: 0.3mm (if specified)
      ```

   c. **Footprint reference** (e.g., cursor on "0402"):
      ```
      **Footprint: 0402**
      Size: 1.0mm x 0.5mm
      Pads: 2
      Type: SMD
      ```

   d. **Pin reference** (e.g., cursor on "R1.1"):
      ```
      **Pin: R1.1**
      Net: VCC
      Component: R1 (resistor)
      ```

3. **Position to byte offset conversion**:
   - Handle UTF-8 properly
   - Use line/column from Position

4. **AST node finding**:
   - Walk AST with spans
   - Find smallest span containing offset

5. **Tests**:
   - Hover on component name
   - Hover on net name
   - Hover on footprint string
   - Hover on whitespace (None)

Convert DRC violations to LSP diagnostics:

1. **run_diagnostics(doc: &DocumentState, client: &Client)**:
   - Build BoardWorld from AST (if not cached)
   - Run DRC with default rules
   - Convert violations to diagnostics
   - Publish via client.publish_diagnostics()

2. **Violation to Diagnostic conversion**:
   - DrcViolation.source_span -> Range (if available)
   - If no span, use document start
   - Severity: DiagnosticSeverity::ERROR (per CONTEXT.md, errors only)
   - Message: violation.message
   - Source: "cypcb-drc"
   - Code: violation.kind as string

3. **Parse errors to Diagnostics**:
   - ParseError also becomes diagnostic
   - Severity: ERROR
   - Source: "cypcb-parser"

4. **Debouncing** (per CONTEXT.md - run on save, not every keystroke):
   - In did_change: set dirty flag
   - In did_save: actually run diagnostics
   - Or: simple approach - run on every change but cap at 100 diagnostics

5. **Diagnostic limit** (per RESEARCH.md pitfall):
   - Max 100 diagnostics per file
   - If more, add "... and N more" summary diagnostic

6. **Integration with backend**:
   - Call run_diagnostics in did_open and did_save
   - Optionally on did_change with debounce

7. **Tests**:
   - Document with clearance violation -> diagnostic
   - Document with parse error -> diagnostic
   - Clean document -> no diagnostics
