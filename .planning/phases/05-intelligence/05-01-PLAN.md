  <action>
Extend the Tree-sitter grammar:

1. **Net constraint properties** (inside net_definition block):
   ```
   net VCC {
       J1.1
       R1.1
       width 0.3mm          // minimum trace width
       clearance 0.2mm      // clearance from other nets
       current 500mA        // for IPC-2221 calculation
   }
   ```
   Grammar additions:
   - net_width_property: "width" dimension
   - net_clearance_property: "clearance" dimension
   - net_current_property: "current" number ("mA" | "A")

2. **Manual trace definition** (new top-level):
   ```
   trace VCC {
       from R1.1
       to C1.1
       via 5mm, 8mm        // optional via waypoints
       layer Top           // copper layer
       width 0.4mm         // override net width
       locked              // prevent autorouter changes
   }
   ```
   Grammar additions:
   - trace_definition: "trace" identifier "{" trace_body "}"
   - trace_body: from_property, to_property, via_waypoint*, layer_property?, width_property?, "locked"?
   - via_waypoint: "via" position_expr

3. **AST types** (in ast.rs):
   - NetConstraints { width: Option<Dimension>, clearance: Option<Dimension>, current: Option<CurrentValue> }
   - CurrentValue { value: f64, unit: CurrentUnit } where CurrentUnit = mA | A
   - TraceDef { name: Identifier, from: PinRef, to: PinRef, waypoints: Vec<PositionExpr>, layer: Option<Identifier>, width: Option<Dimension>, locked: bool }

4. **Parser updates** (in parser.rs):
   - convert_net_constraint() for parsing constraint properties
   - convert_trace_definition() for parsing trace blocks
   - Add Definition::Trace(TraceDef) variant

Run `cd crates/cypcb-parser/grammar && tree-sitter generate` after grammar changes.
