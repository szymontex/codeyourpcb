Implement Specctra SES (Session) file parsing:

1. **SesImportError enum**:
   - IoError(std::io::Error)
   - ParseError(String)
   - NoRoutesFound
   - CoordinateMismatch

2. **import_ses(path: &Path) -> Result<RoutingResult, SesImportError>**:
   - Read SES file content
   - Parse S-expression structure
   - Extract wiring section

3. **SES Structure** (what FreeRouting produces):
   ```
   (session "board"
     (routes
       (network_out
         (net "VCC"
           (wire
             (path Top 8 1000 2000 1500 2000)
           )
           (via via1 1500 2000)
         )
       )
     )
   )
   ```

4. **parse_wire(content: &str) -> Vec<RouteSegment>**:
   - Extract (wire (path layer width x1 y1 x2 y2 ...))
   - Layer: string -> Layer enum
   - Width: mils -> Nm
   - Coordinates: mils -> Nm (multiply by 25400)
   - Create segments from point pairs

5. **parse_via(content: &str) -> Vec<ViaPlacement>**:
   - Extract (via padstack_name x y)
   - Position: mils -> Nm
   - Lookup padstack for drill size and layers

6. **Simple S-expression tokenizer**:
   - Don't need full parser, just extract relevant sections
   - Find (routes ...), then (network_out ...), then each (net ...)
   - Use regex or simple state machine

7. **Coordinate round-trip verification**:
   - DSN exports in mils
   - SES returns in same mils
   - Convert back to nm using same factor (25400)

8. **Tests**:
   - Parse minimal SES with one wire
   - Parse SES with via
   - Parse SES with multiple nets
   - Handle empty routes (no connections routed)

<task type="auto">
  <name>Task 2: Implement FreeRouting CLI Wrapper</name>
  <files>
    crates/cypcb-router/src/freerouting.rs
    crates/cypcb-router/src/lib.rs
  </files>
  <action>
Implement FreeRouting CLI integration (per RESEARCH.md Pattern 2):

1. **RoutingConfig struct**:
   - jar_path: PathBuf (path to freerouting.jar)
   - timeout_secs: u64 (default 300 = 5 minutes)
   - max_passes: Option<u32> (FreeRouting -mp flag)
   - fanout: bool (enable fanout optimization)

2. **FreeRoutingRunner struct**:
   - config: RoutingConfig
   - cancel_flag: Arc<AtomicBool> (for cancellation)

3. **route(&self, dsn_path: &Path, ses_path: &Path) -> Result<RoutingResult>**:
   - Spawn: java -jar freerouting.jar -de input.dsn -do output.ses
   - Monitor process with timeout
   - Check cancel_flag periodically
   - Parse stdout for progress indicators
   - On success: import_ses(ses_path)
   - On timeout/cancel: kill process, return partial if SES exists

4. **Cancellation support**:
   - cancel() method sets cancel_flag
   - Process loop checks flag every 100ms
   - If cancelled, SIGTERM process, return RoutingStatus::Cancelled

5. **Progress parsing** (from FreeRouting stdout):
   - Look for patterns like "Pass N: X connections routed"
   - Emit progress via callback: Box<dyn Fn(RoutingProgress)>
   - RoutingProgress { pass: u32, routed: u32, unrouted: u32 }

6. **Error handling**:
   - Java not found -> RoutingError::JavaNotFound
   - JAR not found -> RoutingError::JarNotFound
   - Process crashed -> RoutingError::ProcessFailed
   - Timeout -> return partial with RoutingStatus::Partial

7. **Tests** (without actual FreeRouting - mock subprocess):
   - Test timeout handling
   - Test cancellation
   - Test progress parsing from mock stdout
  </action>

  </verify>
