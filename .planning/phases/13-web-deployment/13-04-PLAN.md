<task type="auto">
  <name>Task 1: Create deployment configuration files</name>
  <files>.github/workflows/deploy.yml, viewer/public/_headers, viewer/public/_redirects</files>
  <action>
    1. Create .github/workflows/deploy.yml:
       ```yaml
       name: Deploy to Cloudflare Pages
       on:
         push:
           branches: [main, master]
       jobs:
         deploy:
           runs-on: ubuntu-latest
           steps:
             - uses: actions/checkout@v4
             - name: Setup Node
               uses: actions/setup-node@v4
               with:
                 node-version: '20'
             - name: Setup Rust
               uses: dtolnay/rust-toolchain@stable
               with:
                 targets: wasm32-unknown-unknown
             - name: Install wasm-pack
               run: cargo install wasm-pack
             - name: Install dependencies
               run: cd viewer && npm ci
             - name: Build
               run: cd viewer && npm run build:web
             - name: Deploy to Cloudflare Pages
               uses: cloudflare/wrangler-action@v3
               with:
                 apiToken: ${{ secrets.CLOUDFLARE_API_TOKEN }}
                 accountId: ${{ secrets.CLOUDFLARE_ACCOUNT_ID }}
                 command: pages deploy viewer/dist --project-name=codeyourpcb
       ```

    2. Create viewer/public/_headers (Cloudflare Pages custom headers):
       ```
       /*.wasm
         Content-Type: application/wasm
         Cache-Control: public, max-age=31536000, immutable

       /*
         Content-Security-Policy: default-src 'self'; script-src 'self' 'wasm-unsafe-eval'; style-src 'self' 'unsafe-inline'; connect-src 'self' ws: wss:
         X-Content-Type-Options: nosniff
         X-Frame-Options: DENY
       ```

    3. Create viewer/public/_redirects for SPA routing:
       ```
       /*  /index.html  200
       ```

    Do NOT include secrets in the workflow file — they come from GitHub repository secrets.
    Do NOT add wrangler.toml — Cloudflare Pages doesn't need it for static site deployment.
  </action>
  <verify>Files exist and are valid YAML/text. `cd viewer && npm run build:web` still works (public/ files copied to dist/).</verify>
  <done>Deployment config ready. GitHub Actions workflow will build and deploy on push to main.</done>
</task>
