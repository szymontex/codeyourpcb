<task type="auto">
  <name>Task 2: Optimize WASM binary size</name>
  <files>crates/cypcb-render/Cargo.toml, viewer/build-wasm.sh</files>
  <action>
    1. Add release profile to crates/cypcb-render/Cargo.toml (or workspace Cargo.toml if profile is there):
       ```toml
       [profile.release]
       opt-level = "z"       # Optimize for size
       lto = true            # Link-time optimization
       codegen-units = 1     # Better optimization
       panic = "abort"       # Smaller panic handler
       strip = true          # Strip debug symbols
       ```
       Check workspace Cargo.toml first — if profile.release already exists there, add these settings there instead.

    2. Update viewer/build-wasm.sh to run wasm-opt after wasm-pack:
       After the wasm-pack build command, add:
       ```bash
       # Post-build optimization
       if command -v wasm-opt &> /dev/null; then
         echo "Running wasm-opt for size optimization..."
         wasm-opt -O4 --converge \
           viewer/pkg/cypcb_render_bg.wasm \
           -o viewer/pkg/cypcb_render_bg.wasm
         echo "Optimized WASM size:"
         ls -lh viewer/pkg/cypcb_render_bg.wasm
       else
         echo "wasm-opt not found, skipping optimization. Install binaryen for smaller builds."
       fi
       ```

    3. Add `--release` flag to wasm-pack build if not already present (it already has it implicitly via wasm-pack defaults, but verify).

    Do NOT use --enable-simd flag — it reduces browser compatibility.
  </action>
  <verify>Run `cd /workspace/codeyourpcb && ./viewer/build-wasm.sh` — WASM file produced, size reported</verify>
  <done>WASM binary built with size optimizations, wasm-opt post-processing applied when available</done>
</task>
