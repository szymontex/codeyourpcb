<task type="auto">
  <name>Task 1: Create File System Access API wrapper</name>
  <files>viewer/src/file-access.ts</files>
  <action>
    Create viewer/src/file-access.ts with:

    1. `hasFileSystemAccess()`: Returns boolean — checks `'showOpenFilePicker' in window`

    2. `openFile()`: Returns `Promise<{ content: string; name: string; handle: FileSystemFileHandle | null }>`
       - If File System Access API available: call `window.showOpenFilePicker()` with types filter for .cypcb and .ses files. Get file handle, read file, return content + name + handle.
       - If not available: create temporary input element, trigger click, read via FileReader. Return content + name + null handle.
       - Handle user cancellation (AbortError) gracefully — return null.

    3. `saveFile(content: string, handle: FileSystemFileHandle | null, defaultName: string)`: Returns `Promise<FileSystemFileHandle | null>`
       - If handle provided: use handle.createWritable(), write content, close. Return same handle.
       - If no handle but API available: call `window.showSaveFilePicker()` with suggested name. Write content. Return new handle.
       - If API not available: create Blob, generate object URL, trigger download link with defaultName. Return null.

    4. Add TypeScript declarations for showOpenFilePicker/showSaveFilePicker (these are not in default lib.dom.d.ts):
       ```typescript
       declare global {
         interface Window {
           showOpenFilePicker?: (options?: any) => Promise<FileSystemFileHandle[]>;
           showSaveFilePicker?: (options?: any) => Promise<FileSystemFileHandle>;
         }
       }
       ```

    Do NOT use browser-fs-access library — the implementation is simple enough to avoid the dependency.
    Do NOT modify desktop.ts — desktop uses Tauri IPC for file operations, not this module.
  </action>
  <verify>TypeScript compiles without errors: `cd viewer && npx tsc --noEmit`</verify>
  <done>file-access.ts exports openFile and saveFile with File System Access API + fallback</done>
</task>
