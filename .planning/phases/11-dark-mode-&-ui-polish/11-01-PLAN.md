Create `viewer/src/theme/theme-types.ts`:
- Export `Theme = 'light' | 'dark' | 'auto'`
- Export `ResolvedTheme = 'light' | 'dark'`
- Export `ThemeChangeListener = (theme: ResolvedTheme) => void`

Create `viewer/src/theme/theme-manager.ts`:
- Export `ThemeManager` class (singleton pattern)
- Constructor: reads localStorage('theme'), sets up `matchMedia('(prefers-color-scheme: dark)')` listener
- `setTheme(theme: Theme)`: saves to localStorage, calls `updateTheme()`
- `getTheme(): Theme`: returns raw theme preference (light/dark/auto)
- `getResolvedTheme(): ResolvedTheme`: resolves 'auto' using matchMedia
- `subscribe(listener: ThemeChangeListener): () => void`: returns unsubscribe function
- `updateTheme()`: sets `document.documentElement.setAttribute('data-theme', resolved)`, notifies all listeners
- Export a singleton instance: `export const themeManager = new ThemeManager()`

The ThemeManager must handle the `change` event on the matchMedia query so that when the OS switches themes while `auto` is selected, it updates immediately.

Update `viewer/index.html`:
1. Add inline `<script>` in `<head>` BEFORE any CSS:
```html
<script>
  const savedTheme = localStorage.getItem('theme');
  const systemPrefersDark = window.matchMedia('(prefers-color-scheme: dark)').matches;
  const theme = savedTheme === 'light' || savedTheme === 'dark' ? savedTheme : (systemPrefersDark ? 'dark' : 'light');
  document.documentElement.setAttribute('data-theme', theme);
</script>
```
2. Add `<meta name="color-scheme" content="light dark">` in head
3. Keep all existing inline styles but do NOT convert them yet (that's plan 02)

WCAG AA compliance: All text/background pairs must achieve 4.5:1 minimum contrast ratio. The provided values are pre-validated:
- Light: #1a1a1a on #ffffff = 16.7:1 ✓
- Dark: #e0e0e0 on #1e1e1e = 12.5:1 ✓
- Light secondary: #666666 on #ffffff = 5.7:1 ✓
- Dark secondary: #b0b0b0 on #1e1e1e = 8.6:1 ✓

Do NOT use pure black (#000000) backgrounds or pure white (#ffffff) text in dark mode.
