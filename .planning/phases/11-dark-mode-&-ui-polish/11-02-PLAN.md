**viewer/src/layers.ts:**

The `LAYER_COLORS` object currently has hardcoded colors. For PCB-specific colors (copper, silk, violation, ratsnest, via), keep them hardcoded — they represent electrical layers and should NOT change with theme. But for UI colors (background, grid, board_outline), read from CSS custom properties.

Add a function to get theme-dependent colors:

```typescript
/** Get current theme colors by reading CSS custom properties */
export function getThemeColors() {
  const style = getComputedStyle(document.documentElement);
  return {
    background: style.getPropertyValue('--bg-canvas').trim() || '#ffffff',
    grid: style.getPropertyValue('--pcb-grid').trim() || '#e0e0e0',
    board_outline: style.getPropertyValue('--pcb-board-outline').trim() || '#cccc00',
    empty_text: style.getPropertyValue('--pcb-empty-text').trim() || '#666666',
    label: style.getPropertyValue('--pcb-label').trim() || '#333333',
  };
}
```

Keep the existing `LAYER_COLORS` for PCB-specific colors (top_copper, bottom_copper, etc.) — these are domain colors, not theme colors.

**viewer/src/renderer.ts:**

1. Import `getThemeColors` from layers.ts
2. In `render()`, call `const themeColors = getThemeColors()` once at the start
3. Replace `LAYER_COLORS.background` with `themeColors.background` (canvas clear, drill holes)
4. Replace `LAYER_COLORS.grid` with `themeColors.grid` (grid lines)
5. Replace `LAYER_COLORS.board_outline` with `themeColors.board_outline`
6. In `drawEmptyState()`, replace `ctx.fillStyle = '#666'` with `themeColors.empty_text`
7. In `drawComponent()`, replace `ctx.fillStyle = isSelected ? '#FF6600' : '#333'` for the label with `themeColors.label` for the unselected color

Pass `themeColors` as a parameter to `drawGrid`, `drawBoardOutline`, `drawEmptyState`, `drawVia` (for drill hole background), and `drawComponent` (for label color). Or, since `getThemeColors` reads DOM once, call it at render start and thread through.

The key insight: PCB electrical colors (copper red/blue, violation red, ratsnest gold) stay fixed. Only the "canvas environment" (background, grid, labels) changes with theme.
