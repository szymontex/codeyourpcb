  <name>Task 1: Configure updater plugin and file association handling</name>
  <files>

    src-tauri/src/lib.rs
    src-tauri/capabilities/default.json

  <action>
1. **src-tauri/Cargo.toml:** Add `tauri-plugin-updater = "2"` to dependencies.

2. **src-tauri/src/lib.rs:**
   - Register updater plugin: `.plugin(tauri_plugin_updater::Builder::new().build())`
   - Handle file-opened events for .cypcb file association. In the `setup()` callback or via `RunEvent::Opened`, when the app is launched with a file path argument:
     - Check `std::env::args()` for any .cypcb file paths passed as arguments
     - If found, emit a `"file-opened"` event to the frontend with the file path and content
   - This enables double-click .cypcb to open in CodeYourPCB

3. **src-tauri/tauri.conf.json:** Verify bundle section has:
   - `fileAssociations` with ext "cypcb" (should exist from Plan 01)
   - Ensure `bundle.targets` is set to `"all"` for MSI/DMG/AppImage/deb generation

4. **src-tauri/capabilities/default.json:** Add `"updater:default"` to permissions array.

5. **Updater endpoint:** Configure a placeholder updater endpoint in tauri.conf.json under `plugins.updater`:
   ```json
   "plugins": {
     "updater": {
       "endpoints": ["https://releases.codeyourpcb.com/{{target}}/{{arch}}/{{current_version}}"],
       "pubkey": ""
     }
   }
   ```
   Note: The pubkey and actual endpoint will be configured when releases infrastructure exists. For now, the updater will gracefully fail to check (no crash).
  </action>
  <verify>
`cargo check -p cypcb-desktop` compiles with updater plugin. tauri.conf.json is valid JSON with updater and fileAssociations sections. Verify updater plugin is wired: `grep "tauri_plugin_updater" src-tauri/src/lib.rs` returns a match.
  </verify>
  <done>Updater plugin registered, file association configured in bundle, file-opened events handled for double-click launch.</done>
