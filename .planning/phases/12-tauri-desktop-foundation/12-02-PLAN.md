<task type="auto">
  <name>Task 1: Create native menu bar from platform MenuBar data model</name>
  <files>src-tauri/src/menu.rs</files>
  <action>
Create `src-tauri/src/menu.rs` with:

1. **`create_app_menu() -> MenuBar`** - Build the application menu using `cypcb_platform::{MenuBar, Menu, MenuItem}`:
   - **File:** New (Ctrl+N), Open... (Ctrl+O), separator, Save (Ctrl+S), Save As... (Ctrl+Shift+S), separator, Quit (Ctrl+Q)
   - **Edit:** Undo (Ctrl+Z), Redo (Ctrl+Shift+Z), separator, Cut (Ctrl+X), Copy (Ctrl+C), Paste (Ctrl+V)
   - **View:** Zoom In (Ctrl+=), Zoom Out (Ctrl+-), Fit to Window (Ctrl+0), separator, Toggle Fullscreen (F11), separator, Toggle Theme (Ctrl+Shift+T)
   - **Help:** About CodeYourPCB

2. **`build_tauri_menu(app: &tauri::App, menu_bar: &MenuBar) -> tauri::Result<tauri::menu::Menu<tauri::Wry>>`** - Convert the platform MenuBar into Tauri native menu:
   - Iterate `menu_bar.items`, create `SubmenuBuilder` for each `Menu`
   - For each `MenuItem::Action`, create `MenuItem::with_id()` passing id, label, enabled, and accelerator string
   - For `MenuItem::Separator`, call `.separator()`
   - For `MenuItem::Submenu`, recurse
   - Shortcut translation: Replace "Ctrl+" with "CmdOrCtrl+" for cross-platform compatibility

3. **`handle_menu_event(app_handle: &tauri::AppHandle, event: &tauri::menu::MenuEvent)`** - Handle menu clicks:
   - Match on `event.id().as_ref()`:
     - "file.quit" -> `app_handle.exit(0)`
     - "view.fullscreen" -> toggle fullscreen on focused window
     - All other IDs -> emit `"menu-action"` event to frontend with the menu ID as payload string
   - This allows the frontend to handle File Open, Save, theme toggle, etc.

**DESK-04 (window management):** Minimize, maximize, and close use native OS titlebar controls (Tauri windows have standard titlebar by default). Fullscreen toggle is handled via the View menu item above. No additional implementation needed for minimize/maximize.
  </action>

`cargo check -p cypcb-desktop` compiles. The menu module is declared in lib.rs with `mod menu;`.
  </verify>
  <done>Native menu bar renders File/Edit/View/Help with keyboard shortcuts. Menu events are forwarded to frontend via Tauri events.</done>

**src-tauri/src/commands.rs:**

All command functions MUST be `pub` and annotated with `#[tauri::command]`. The module itself must be declared as `pub mod commands;` in lib.rs so the `generate_handler!` macro can access them.

1. **`open_file(app: tauri::AppHandle) -> Result<Option<FileContent>, String>`** - Tauri command:
   - Use `tauri_plugin_dialog::DialogExt` to open a file picker
   - Filter: "CodeYourPCB" with extension `["cypcb"]`
   - If user selects file, read content with `std::fs::read_to_string`
   - Return `FileContent { path: String, content: String }` (define this struct with Serialize)
   - If user cancels, return `Ok(None)`

2. **`save_file(path: String, content: String) -> Result<(), String>`** - Tauri command:
   - Write content to the given path using `std::fs::write`
   - Return Ok on success, Err with message on failure

3. **`save_file_as(app: tauri::AppHandle, content: String) -> Result<Option<String>, String>`** - Tauri command:
   - Use dialog to show save file picker with .cypcb filter
   - Write content to selected path
   - Return the chosen path or None if cancelled

4. Define `FileContent` struct with `Serialize`:
   ```rust
   #[derive(serde::Serialize)]
   pub struct FileContent {
       pub path: String,
       pub content: String,
   }
   ```

**src-tauri/src/lib.rs:**
- Add `mod menu;` and `pub mod commands;` declarations (commands must be pub for generate_handler! macro)
- In the `run()` function's `setup()` callback:
  - Call `menu::create_app_menu()` to get the MenuBar
  - Call `menu::build_tauri_menu(app, &menu_bar)` to create native menu
  - Set the menu on the app with `app.set_menu(tauri_menu)?`
  - Register menu event handler: `app.on_menu_event(|app_handle, event| menu::handle_menu_event(app_handle, &event))`
- Register invoke handler: `tauri::generate_handler![commands::open_file, commands::save_file, commands::save_file_as]`
