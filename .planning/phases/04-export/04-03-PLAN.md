  <files>
    crates/cypcb-export/src/gerber/outline.rs
    crates/cypcb-export/src/gerber/mod.rs
  </files>
  <action>
Implement board outline/profile Gerber export:

1. Create crates/cypcb-export/src/gerber/outline.rs
2. Update mod.rs to include `pub mod outline;`

3. Define export_outline function:
   ```rust
   pub fn export_outline(
       world: &BoardWorld,
       format: &CoordinateFormat,
   ) -> Result<String, ExportError>
   ```

4. Get board dimensions:
   - Call world.board_info() to get BoardSize
   - Calculate corner points: (0,0), (w,0), (w,h), (0,h)
   - For simple rectangular boards, this is the complete outline

5. Generate outline Gerber:
   - Header with TF.FileFunction,Profile,NP (Non-Plated profile)
   - Create thin line aperture (0.1mm default width)
   - Draw closed polygon:
     - D02* move to first corner (0, 0)
     - D01* draw to (width, 0)
     - D01* draw to (width, height)
     - D01* draw to (0, height)
     - D01* draw back to (0, 0) to close
   - M02* end of file

6. Handle zones with ZoneKind::BoardOutline (if present):
   - Check for Zone entities with ZoneKind::BoardOutline
   - If found, use zone bounds instead of board size
   - This allows non-rectangular boards

7. Add route tool width constant:
   - OUTLINE_WIDTH: Nm = Nm::from_mm(0.1)
   - This matches typical router bit kerf

8. Add tests:
   - Test rectangular board outline
   - Test output is closed polygon
   - Test correct coordinates at corners
   - Verify Profile file function in header

Note: Profile layer uses lines (D01) not flashes (D03). The path must be CLOSED (return to start point) for manufacturing.
  </action>
  <verify>cargo test -p cypcb-export gerber::outline passes</verify>
  <done>Board outline exports as closed Gerber polygon</done>
</task>

3. Define export_silkscreen function:
   ```rust
   pub fn export_silkscreen(
       world: &mut BoardWorld,
       library: &FootprintLibrary,
       side: Side,  // Top or Bottom
       format: &CoordinateFormat,
   ) -> Result<String, ExportError>
   ```

4. Silkscreen content for MVP:
   - Component reference designators (R1, C1, U1)
   - Component outlines from footprint courtyard

5. Implement designator rendering:
   - For each component on target side:
     - Get RefDes component
     - Calculate text position (component center + offset)
     - Generate text as line segments (simple stroke font)

   NOTE: Gerber has no native text support. Options:
   a) Skip text for MVP (outline only)
   b) Render text as polylines (complex)
   c) Use simple placeholder marks

   DECISION: For MVP, render designator as simple crosshair (+) mark at component center to indicate component location without complex text rendering.

6. Implement component outline:
   - Get footprint courtyard bounds
   - Translate to component position
   - Draw courtyard rectangle outline

7. Add silkscreen configuration:
   ```rust
   pub struct SilkConfig {
       pub line_width: Nm,  // Default 0.15mm
       pub show_courtyards: bool,
       pub show_designator_marks: bool,
   }
   ```

8. Generate complete file:
   - Header with TF.FileFunction,Legend,{Top|Bot}
   - Aperture for line width
   - Courtyard outlines as rectangles
   - Designator marks as crosshairs
   - M02* end

9. Add tests:
   - Test silkscreen has content for components
   - Test correct side filtering
   - Test line aperture definition

Note: Full text rendering is complex and deferred. MVP uses markers. Real silkscreen text would require either bitmap fonts or vector stroke fonts converted to Gerber polylines.
