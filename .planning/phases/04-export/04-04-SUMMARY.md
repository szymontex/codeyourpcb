### Task 2: Excellon Drill File Export

**Deliverables:**
- `export_excellon(world, library, format)` generates complete Excellon files
- `collect_drill_hits()` extracts drills from:
  - Component pads with `drill.is_some()` (through-hole pads)
  - Via entities (via.drill)
- `calculate_pad_position()` handles component rotation via trigonometry
- `group_hits_by_tool()` minimizes tool changes by grouping
- File structure:
  ```
  M48                          ; Start of header
  ; DRILL file generated by CodeYourPCB
  ; Plated Through Holes
  METRIC,TZ                    ; Metric units, trailing zero suppression
  T1C0.3000                    ; Tool definitions
  T2C0.8000
  %                            ; End of header
  T1                           ; Select tool
  X50.800Y30.480               ; Drill hit coordinates
  X55.880Y30.480
  T2
  X63.500Y50.800
  M30                          ; End of file
  ```

**Implementation Highlights:**
```rust
fn calculate_pad_position(component_pos: Point, pad_offset: Point, rotation_millideg: i32) -> Point {
    if rotation_millideg == 0 {
        // No rotation: simple addition
        Point::new(
            Nm(component_pos.x.0 + pad_offset.x.0),
            Nm(component_pos.y.0 + pad_offset.y.0),
        )
    } else {
        // Rotate pad offset, then add to component position
        let angle_rad = (rotation_millideg as f64) / 1000.0 * PI / 180.0;
        let cos = angle_rad.cos();
        let sin = angle_rad.sin();
        let rotated_x = pad_offset.x.0 as f64 * cos - pad_offset.y.0 as f64 * sin;
        let rotated_y = pad_offset.x.0 as f64 * sin + pad_offset.y.0 as f64 * cos;
        // ...
    }
}
```

### Task 3: Plated vs Non-Plated Drill Separation

**Deliverables:**
- `DrillType` enum: `Plated` (PTH), `NonPlated` (NPTH)
- Extended `export_excellon()` with `drill_type_filter: Option<DrillType>`
- Filter options:
  - `None` - Export all drills
  - `Some(DrillType::Plated)` - PTH only (component pads, vias)
  - `Some(DrillType::NonPlated)` - NPTH only (future mounting holes)
- Header comments identify drill type:
  - PTH: `; Plated Through Holes`
  - NPTH: `; Non-Plated Through Holes`
  - All: `; All drill holes`
- `group_hits_by_tool_refs()` handles reference-based grouping after filtering
- Empty file generation customized by drill type

**Implementation Highlights:**
```rust
pub enum DrillType {
    Plated,      // PTH - component holes, vias
    NonPlated,   // NPTH - mounting holes
}

struct DrillHit {
    position: Point,
    drill_diameter: Nm,
    drill_type: DrillType,  // Added for filtering
}

// Filter during export
let hits: Vec<&DrillHit> = if let Some(filter_type) = drill_type_filter {
    all_hits.iter().filter(|h| h.drill_type == filter_type).collect()
} else {
    all_hits.iter().collect()
};
```

**Current Behavior (MVP):**
- All component pad drills → `DrillType::Plated`
- All vias → `DrillType::Plated`
- NPTH support is infrastructure for future mounting hole zones

## Known Limitations

1. **NPTH stub only:** No mounting hole zone implementation yet
   - All drills currently Plated (pads, vias)
   - Infrastructure ready for future NPTH support

2. **No blind/buried via support:** Assumes through-hole vias
   - `Via.start_layer` and `Via.end_layer` exist but not used in export
   - Excellon format supports buried vias, implementation deferred

3. **Single coordinate format:** Uses CoordinateFormat from 04-01
   - Could support inch format, but metric is industry standard
   - No demand for inch format in MVP
