<task type="auto">
  <name>Task 1: Export example board and validate structure</name>
  <files></files>
  <action>
Export the blink.cypcb example and validate file structure:

1. Run export command:
   ```bash
   cargo run -- export examples/blink.cypcb -o /tmp/blink-export
   ```

2. Verify output structure exists:
   ```
   /tmp/blink-export/
   ├── gerber/
   │   ├── blink-F_Cu.gbr      (top copper)
   │   ├── blink-B_Cu.gbr      (bottom copper)
   │   ├── blink-F_Mask.gbr    (top soldermask)
   │   ├── blink-B_Mask.gbr    (bottom soldermask)
   │   ├── blink-F_SilkS.gbr   (top silkscreen)
   │   ├── blink-B_SilkS.gbr   (bottom silkscreen)
   │   ├── blink-F_Paste.gbr   (top paste)
   │   ├── blink-B_Paste.gbr   (bottom paste)
   │   └── blink-Edge_Cuts.gbr (outline)
   ├── drill/
   │   ├── blink-PTH.drl       (plated through holes)
   │   └── blink-NPTH.drl      (non-plated, if any)
   └── assembly/
       ├── blink-BOM.csv
       ├── blink-BOM.json
       └── blink-CPL.csv
   ```

3. Verify non-empty files:
   - All Gerber files > 100 bytes (header alone)
   - Drill file has at least header if THT components exist
   - BOM/CPL have headers + data rows

4. Spot-check Gerber header format:
   - Has %FSLAX format line
   - Has %MOMM unit line
   - Has TF.FileFunction attribute

5. Spot-check Excellon header:
   - Has M48 start
   - Has METRIC declaration
   - Has tool definitions if drills present

6. Spot-check BOM CSV:
   - Has Designator column
   - Has Footprint column
   - Component count matches design

7. Spot-check CPL CSV:
   - Has Mid X, Mid Y columns
   - Coordinates are reasonable (positive, within board)
  </action>
  <verify>All files exist and have valid structure</verify>
  <done>Export produces complete file set with valid structure</done>
</task>

<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
Complete export pipeline generating Gerber, Excellon, BOM, and CPL files from .cypcb source.
  </what-built>
  <how-to-verify>
1. **Validate Gerber files with gerbv or online viewer:**
   - Open `/tmp/blink-export/gerber/blink-F_Cu.gbr` in https://gerber-viewer.ucamco.com/ or local gerbv
   - Verify pads are visible at expected locations
   - Load multiple layers and verify they align (pads, outline)

2. **Check Excellon drill alignment:**
   - Load drill file in same viewer
   - Verify drill holes appear at center of THT pads
   - Check that drill sizes look reasonable

3. **Review BOM content:**
   - Open `blink-BOM.csv` in text editor or spreadsheet
   - Verify all expected components listed (R1, LED1, etc.)
   - Check quantities are correct (identical parts grouped)
   - Verify values and footprints match design

4. **Review CPL content:**
   - Open `blink-CPL.csv`
   - Verify coordinates are in millimeters
   - Check rotation values are reasonable (0, 90, 180, 270 typically)
   - Verify layer column shows Top or Bottom correctly

5. **Optional: JLCPCB web upload test:**
   - Go to https://cart.jlcpcb.com/quote
   - Upload the gerber folder as ZIP
   - Check if DFM preview accepts files
   - (Don't need to complete order, just verify acceptance)
