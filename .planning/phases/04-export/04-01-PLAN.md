<task type="auto">
  <name>Task 2: Implement coordinate conversion</name>
  <files>crates/cypcb-export/src/coords.rs</files>
  <action>
Create coordinate conversion module for Gerber/Excellon output:

1. Define CoordinateFormat struct:
   - unit: Unit enum (Millimeters, Inches)
   - integer_places: u8 (typically 2-4)
   - decimal_places: u8 (typically 4-6)

2. Implement nm_to_gerber(nm: i64, format: &CoordinateFormat) -> String:
   - Use integer arithmetic to avoid floating-point precision loss
   - For mm: 1mm = 1_000_000 nm
   - For inches: 1 inch = 25_400_000 nm
   - Format with correct decimal places, zero-padding as needed
   - Handle negative coordinates correctly

3. Implement gerber_format_string(format: &CoordinateFormat) -> String:
   - Return %FSLA{X}{int}{dec}Y{int}{dec}*% format string
   - Example: %FSLAX26Y26*% for 2 integer, 6 decimal places

4. Add constants for common formats:
   - FORMAT_MM_2_6: 2 integer, 6 decimal (mm) - most common
   - FORMAT_INCH_2_4: 2 integer, 4 decimal (inches) - legacy

5. Add comprehensive tests:
   - Test 0nm -> "0.000000"
   - Test 1_000_000nm (1mm) -> "1.000000"
   - Test 25_400_000nm (1 inch) -> "1.0000" (inches format)
   - Test negative coordinates
   - Test precision edge cases

AVOID: Using f64 intermediate values - go directly from i64 nm to decimal string.
  </action>
  <verify>cargo test -p cypcb-export coords passes all tests</verify>
  <done>Coordinate conversion handles nm to Gerber decimal format correctly</done>
</task>

<task type="auto">
  <name>Task 3: Implement aperture management</name>
  <files>crates/cypcb-export/src/apertures.rs</files>
  <action>
Create aperture management for Gerber D-codes:

1. Define ApertureShape enum matching pad shapes:
   - Circle { diameter: i64 } (in nm)
   - Rectangle { width: i64, height: i64 }
   - Oblong { width: i64, height: i64 }
   - RoundRect { width: i64, height: i64, corner_ratio: u8 }

2. Implement ApertureManager struct:
   - next_dcode: u16 (starts at 10 per Gerber convention)
   - apertures: HashMap<ApertureShape, u16>

3. Implement ApertureManager methods:
   - new() -> Self
   - get_or_create(shape: ApertureShape) -> u16
     Returns existing D-code or creates new one
   - to_definitions(&self, format: &CoordinateFormat) -> String
     Generates all %ADD...% definition lines

4. Implement aperture_for_pad(pad: &PadDef, format: &CoordinateFormat) -> ApertureShape:
   - Map PadShape::Circle -> ApertureShape::Circle
   - Map PadShape::Rect -> ApertureShape::Rectangle
   - Map PadShape::Oblong -> ApertureShape::Oblong
   - Map PadShape::RoundRect -> ApertureShape::RoundRect
   - Use pad size from PadDef

5. Implement aperture definition formatting:
   - Circle: %ADD{d}C,{diameter}*%
   - Rectangle: %ADD{d}R,{width}X{height}*%
   - Oblong: %ADD{d}O,{width}X{height}*%
   - RoundRect: needs polygon approximation or rectangle (fallback)

6. Add tests:
   - Test aperture reuse (same shape returns same D-code)
   - Test definition string generation
   - Test all pad shape conversions

Note: D-codes start at 10 (D10, D11, ...) per Gerber standard. D01-D03 are reserved for draw/move/flash commands.
  </action>
  <verify>cargo test -p cypcb-export apertures passes all tests</verify>
