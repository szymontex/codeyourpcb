<task type="auto">
  <name>Task 2: Implement Excellon drill file export</name>
  <files>crates/cypcb-export/src/excellon/writer.rs</files>
  <action>
Implement Excellon drill file generation:

1. Define DrillHit struct:
   ```rust
   struct DrillHit {
       position: Point,
       drill_diameter: Nm,
   }
   ```

2. Implement collect_drill_hits function:
   - Parameters: world, library
   - Returns: Vec<DrillHit>
   - Collect from THT pads:
     - Query components with FootprintRef
     - Look up footprint in library
     - For each pad with drill.is_some():
       - Calculate absolute position (component + rotated pad offset)
       - Add DrillHit with drill diameter
   - Collect from vias:
     - Query Via components
     - Add DrillHit with via.drill

3. Implement export_excellon function:
   ```rust
   pub fn export_excellon(
       world: &mut BoardWorld,
       library: &FootprintLibrary,
       format: &CoordinateFormat,
   ) -> Result<String, ExportError>
   ```

4. Generate Excellon file structure:
   ```
   M48                          ; Start of header
   ; DRILL file generated by CodeYourPCB
   ; FORMAT={2:4/ absolute / metric / suppress trailing zeros}
   METRIC,TZ                    ; Metric units, trailing zero suppression
   T1C0.3000                    ; Tool 1 = 0.3mm
   T2C0.8000                    ; Tool 2 = 0.8mm
   %                            ; End of header
   T1                           ; Select tool 1
   X50.800Y30.480               ; Drill hit coordinates
   X55.880Y30.480
   T2                           ; Select tool 2
   X63.500Y50.800
   M30                          ; End of file
   ```

5. Coordinate formatting for Excellon:
   - Use METRIC mode (most common)
   - Format: X{mm}Y{mm} without decimal point (implicit)
   - Or with decimal: X50.800Y30.480 (explicit)
   - Use explicit decimal for clarity

6. Group drill hits by tool:
   - Sort hits by drill diameter
   - Output T{n} command then all hits for that tool
   - Minimizes tool changes

7. Add tests:
   - Test header format
   - Test coordinate output
   - Test multiple tool sizes
   - Test empty board (no drills)
   - Test via drill collection

CRITICAL: Excellon header MUST declare units (METRIC or INCH). Missing header = manufacturer guesses = wrong holes.
  </action>
  <verify>cargo test -p cypcb-export excellon::writer passes</verify>
  <done>Excellon export generates valid drill file with header and coordinates</done>
