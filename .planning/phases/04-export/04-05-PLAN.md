Output: BOM export in CSV/JSON formats, CPL export in CSV format matching JLCPCB requirements.

1. Create crates/cypcb-export/src/cpl/mod.rs:
   - pub mod csv;

2. Update lib.rs to include `pub mod cpl;`

3. Define CplEntry struct:
   ```rust
   #[derive(Debug, Clone, Serialize)]
   pub struct CplEntry {
       pub designator: String,    // "U1"
       pub x_mm: f64,             // 50.800 (center X in mm)
       pub y_mm: f64,             // 30.480 (center Y in mm)
       pub layer: String,         // "Top" or "Bottom"
       pub rotation: f64,         // 0.0 to 359.999 degrees
   }
   ```

4. Implement component center calculation:
   - Component position is already the center (component origin)
   - Convert from Nm to mm for output
   - Account for rotation in degrees (from millidegrees)

5. Implement layer detection:
   - Determine if component is on top or bottom
   - Check first pad's layer in footprint
   - Or use convention: assume top unless explicitly bottom

6. In cpl/csv.rs, implement export_cpl:
   ```rust
   pub fn export_cpl(
       world: &mut BoardWorld,
       library: &FootprintLibrary,
   ) -> Result<String, ExportError>
   ```

7. Define CSV row struct matching JLCPCB format:
   ```rust
   #[derive(Serialize)]
   struct CplCsvRow {
       #[serde(rename = "Designator")]
       designator: String,
       #[serde(rename = "Mid X")]
       mid_x: String,  // "50.800mm"
       #[serde(rename = "Mid Y")]
       mid_y: String,  // "30.480mm"
       #[serde(rename = "Layer")]
       layer: String,
       #[serde(rename = "Rotation")]
       rotation: String,  // "90"
   }
   ```

8. Handle rotation:
   - Internal: millidegrees (0-359999)
   - Output: degrees (0-359.999)
   - Conversion: millidegrees / 1000.0
   - Convention: counterclockwise from 0 (3 o'clock)

9. Add rotation offset parameter for manufacturer adjustment:
   ```rust
   pub struct CplConfig {
       pub rotation_offset: f64,  // Added to all rotations
       pub flip_y: bool,          // Some machines use Y-down
   }
   ```

10. Add tests:
    - Test coordinate conversion (nm to mm)
    - Test rotation conversion (millidegrees to degrees)
    - Test layer detection
    - Test CSV output format
    - Test all components included

Note: Pick-and-place coordinates are component CENTER, not corner. This is different from some other coordinate references.
