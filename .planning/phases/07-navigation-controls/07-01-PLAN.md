  <name>Task 1: Add touch-action CSS and Pointer Events multi-touch pan</name>
  <files>viewer/src/interaction.ts, viewer/index.html</files>
  <action>
1. In viewer/index.html, add `touch-action: none;` to the canvas CSS rule. This prevents the browser from hijacking touch/pointer gestures on the canvas element. Must be in CSS, not JavaScript, for compositor thread performance.

2. In viewer/src/interaction.ts, add a pointer cache and Pointer Events handlers inside setupInteraction():

   a. Define a local `pointerCache` array of `{ pointerId: number; clientX: number; clientY: number }` objects.

   b. Add `pointerdown` listener: push pointer to cache (`{ pointerId: e.pointerId, clientX: e.clientX, clientY: e.clientY }`).

   c. Add `pointermove` listener: find pointer in cache by `e.pointerId`. If found AND `pointerCache.length === 2`:
      - Calculate dx/dy from cached position to current position
      - Call `state.viewport = pan(state.viewport, dx / 2, dy / 2)` (half delta since both fingers contribute)
      - Set `state.dirty = true` and call `state.onViewportChange(state.viewport)`
      - Update the cached pointer entry with new clientX/clientY
      If found but cache length is NOT 2, just update the cached entry (single pointer is handled by existing mousedown/mousemove).

   d. Create a shared cleanup function `removePointer(e: PointerEvent)` that splices the matching pointerId from cache.

   e. Add cleanup listeners for: `pointerup`, `pointercancel`, `pointerout`, `pointerleave` - all call `removePointer`.

3. Important: Do NOT remove or change existing mousedown/mousemove/mouseup handlers. The pointer events handle multi-touch ONLY (2+ fingers). Single-pointer interactions continue through the existing mouse event handlers. This ensures middle-click pan, Ctrl+LMB pan, and left-click selection all remain unchanged.

4. Do NOT use Touch Events (legacy). Use Pointer Events only (modern standard, baseline since July 2020).
