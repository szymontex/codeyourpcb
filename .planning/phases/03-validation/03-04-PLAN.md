<task type="auto">
  <name>Task 1: Add footprint grammar rules</name>
  <files>crates/cypcb-parser/grammar/grammar.js</files>
  <action>
Extend the Tree-sitter grammar to support footprint definitions.

Add to grammar.js:

1. Add `footprint_definition` to top-level definition alternatives.

2. Create footprint_definition rule:
   ```javascript
   footprint_definition: $ => seq(
     'footprint',
     $.identifier,
     '{',
     repeat($.footprint_property),
     '}'
   ),
   ```

3. Create footprint_property choice:
   ```javascript
   footprint_property: $ => choice(
     $.description_property,   // description "text"
     $.pad_definition,         // pad N shape at X, Y size W x H
     $.courtyard_property,     // courtyard W x H (optional)
   ),
   ```

4. Create pad_definition rule:
   ```javascript
   pad_definition: $ => seq(
     'pad',
     $.number,                 // Numeric pad ID (1, 2, 3...)
     $.pad_shape,              // rect, circle, roundrect, oblong
     'at',
     $.dimension,              // X position
     ',',
     $.dimension,              // Y position
     'size',
     $.dimension,              // Width
     'x',
     $.dimension,              // Height
     optional($.drill_spec),   // Optional: drill Nmm
   ),

   drill_spec: $ => seq('drill', $.dimension),

   pad_shape: $ => choice('rect', 'circle', 'roundrect', 'oblong'),
   ```

5. Create courtyard_property rule:
   ```javascript
   courtyard_property: $ => seq(
     'courtyard',
     $.dimension,
     'x',
     $.dimension
   ),
   ```

6. Regenerate parser: `cd grammar && npx tree-sitter generate`

Target syntax (from 03-RESEARCH.md):
```cypcb
footprint MY_SOIC_8 {
    description "Custom SOIC-8 with thermal pad"
    pad 1 rect at -2.7mm, -1.905mm size 1.5mm x 0.6mm
    pad 2 rect at -2.7mm, -0.635mm size 1.5mm x 0.6mm
    // ...
    courtyard 6mm x 5mm
}
```

<task type="auto">
  <name>Task 2: Add FootprintDef and PadDef AST types</name>
  <files>crates/cypcb-parser/src/ast.rs</files>
  <action>
Add AST types to ast.rs:

1. PadDef struct:
   ```rust
   /// A pad definition within a footprint.
   #[derive(Debug, Clone, Serialize, Deserialize)]
   pub struct PadDef {
       /// Pad number (e.g., 1, 2, 3).
       pub number: u32,
       /// Pad shape.
       pub shape: PadShape,
       /// X position relative to footprint origin.
       pub x: Dimension,
       /// Y position relative to footprint origin.
       pub y: Dimension,
       /// Pad width.
       pub width: Dimension,
       /// Pad height.
       pub height: Dimension,
       /// Optional drill diameter for through-hole pads.
       pub drill: Option<Dimension>,
       /// Source span.
       pub span: Span,
   }
   ```

2. PadShape enum:
   ```rust
   #[derive(Debug, Clone, Copy, PartialEq, Eq, Serialize, Deserialize)]
   pub enum PadShape {
       Rect,
       Circle,
       RoundRect,
       Oblong,
   }
   ```

3. FootprintDef struct:
   ```rust
   /// A custom footprint definition.
   #[derive(Debug, Clone, Serialize, Deserialize)]
   pub struct FootprintDef {
       /// Footprint name/identifier.
       pub name: Identifier,
       /// Optional description.
       pub description: Option<String>,
       /// Pad definitions.
       pub pads: Vec<PadDef>,
       /// Optional explicit courtyard dimensions.
       pub courtyard: Option<(Dimension, Dimension)>,
       /// Source span.
       pub span: Span,
   }
   ```

4. Add FootprintDef variant to Definition enum:
   ```rust
   pub enum Definition {
       Board(BoardDef),
       Component(ComponentDef),
       Net(NetDef),
       Footprint(FootprintDef),  // NEW
   }
   ```
