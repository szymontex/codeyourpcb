<task type="auto">
  <name>Task 2: Create SQLite schema with FTS5 and component CRUD</name>
  <files>
    crates/cypcb-library/src/schema.rs
  </files>
  <action>
Create `schema.rs` with:

1. A constant `LIBRARY_SCHEMA: &str` containing SQL to create:
   - `libraries` table: source TEXT, name TEXT, path TEXT, version TEXT, enabled INTEGER DEFAULT 1, component_count INTEGER DEFAULT 0, PRIMARY KEY (source, name)
   - `components` table: rowid INTEGER PRIMARY KEY AUTOINCREMENT, source TEXT NOT NULL, name TEXT NOT NULL, library TEXT NOT NULL, category TEXT, footprint_data TEXT, description TEXT, datasheet_url TEXT, manufacturer TEXT, mpn TEXT, value TEXT, package TEXT, step_model_path TEXT, metadata_json TEXT, UNIQUE(source, name), FOREIGN KEY (source, library) REFERENCES libraries(source, name)
   - `components_fts` FTS5 virtual table: source, name, category, description, manufacturer, mpn, value, package, content=components, content_rowid=rowid
   - INSERT/DELETE/UPDATE triggers to sync components_fts with components table (see research Pattern 2)
   - Indexes on components(category), components(manufacturer), components(value)

2. Function `pub fn initialize_schema(conn: &rusqlite::Connection) -> Result<(), LibraryError>` that executes LIBRARY_SCHEMA using `conn.execute_batch()`.

3. CRUD functions (all synchronous, taking &Connection):
   - `pub fn insert_library(conn: &Connection, lib: &LibraryInfo) -> Result<(), LibraryError>`
   - `pub fn list_libraries(conn: &Connection) -> Result<Vec<LibraryInfo>, LibraryError>`
   - `pub fn insert_component(conn: &Connection, component: &Component) -> Result<(), LibraryError>` - inserts into components table, FTS5 syncs via trigger
   - `pub fn insert_components_batch(conn: &Connection, components: &[Component]) -> Result<usize, LibraryError>` - uses transaction for batch insert, returns count inserted
   - `pub fn get_component(conn: &Connection, source: &str, name: &str) -> Result<Option<Component>, LibraryError>`
   - `pub fn delete_library_components(conn: &Connection, source: &str, library: &str) -> Result<usize, LibraryError>` - deletes all components for a library, returns count

Use rusqlite `params!` macro for parameterized queries (NOT string formatting - SQL injection risk).
  </action>
