Add `lexpr = "0.2.7"` to cypcb-library Cargo.toml.

Create `sources/mod.rs` with:
- `LibrarySource` trait (NOT async - keep it simple, use blocking I/O since this runs in spawn_blocking):
  ```rust
  pub trait LibrarySource {
      fn source_name(&self) -> &str;
      fn list_libraries(&self) -> Result<Vec<LibraryInfo>, LibraryError>;
      fn import_library(&self, name: &str) -> Result<Vec<Component>, LibraryError>;
  }
  ```
- `pub mod kicad;`

Create `sources/kicad.rs` with:
- `KiCadSource` struct with field `search_paths: Vec<PathBuf>`
- Constructor `new(search_paths: Vec<PathBuf>) -> Self`
- `impl LibrarySource for KiCadSource`:
  - `source_name()` returns `"kicad"`
  - `list_libraries()`: Scans each search_path for subdirectories ending in `.pretty`. Returns LibraryInfo with source="kicad", name=folder_name (without .pretty suffix), path=full_path.
  - `import_library()`: Finds the .pretty folder matching `name`, reads all `.kicad_mod` files within it, parses each with `parse_kicad_mod()`, returns Vec<Component>.

- `fn parse_kicad_mod(path: &Path) -> Result<Component, LibraryError>`:
  - Read file contents with `std::fs::read_to_string`
  - Parse with `lexpr::from_str(&content)`
  - Extract from S-expression tree:
    - Component name from `(footprint "NAME" ...)` - first string after "footprint" token
    - Description from `(descr "...")` if present
    - Layer from `(layer "...")` if present
    - Pad data for basic footprint characterization
  - Do NOT try to use Serde deserialization with lexpr (KiCad S-expressions are not standard Serde-compatible). Instead, manually walk the `lexpr::Value` tree:
    - `Value::Cons` for list traversal
    - `Value::Symbol` for keywords like "footprint", "descr", "layer"
    - `Value::String` for quoted values
  - Build `Component` with `source="kicad"`, extracted name, category derived from layer (e.g., "SMD" for F.Cu), metadata from description/layer.
  - Store the raw S-expression content in `footprint_data` field for later preview rendering.

- `fn auto_organize_folder(path: &Path) -> Result<Vec<LibraryInfo>, LibraryError>`:
  - If path is a .pretty folder, treat it as a single library
  - If path is a directory containing .pretty folders, treat each as a library
  - Returns list of discovered libraries (for LIB-11 "idiot-proof" drop)

Update `lib.rs` to add `pub mod sources;`.

IMPORTANT: lexpr's Value type uses Cons cells (Lisp-style linked lists), NOT Vec. Navigate with pattern matching on Value::Cons(car, cdr). The research examples showing Serde deserialization with lexpr may not work for KiCad's format - use manual tree walking instead.
