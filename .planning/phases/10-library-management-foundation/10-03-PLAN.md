  <action>
Create `search.rs` with:

1. `pub fn search_components(conn: &Connection, query: &str, filters: &SearchFilters) -> Result<Vec<SearchResult>, LibraryError>`:
   - Sanitize query: escape double quotes, trim whitespace. If empty, return empty vec.
   - Build FTS5 MATCH query. For plain text like "resistor 0805", convert to FTS5 syntax: each word becomes a term, joined implicitly (AND semantics).
   - Support prefix matching: if query ends with *, pass through to FTS5.
   - Build SQL:
     ```sql
     SELECT c.source, c.name, c.library, c.category, c.footprint_data,
            c.description, c.datasheet_url, c.manufacturer, c.mpn,
            c.value, c.package, c.step_model_path, c.metadata_json,
            bm25(components_fts) as rank
     FROM components c
     JOIN components_fts fts ON c.rowid = fts.rowid
     WHERE components_fts MATCH ?1
     ```
   - Append optional WHERE clauses from filters (source, category, manufacturer) using AND with parameterized queries.
   - Append `ORDER BY rank LIMIT ?` using filters.limit.
   - Map rows to SearchResult structs.
   - Use `params![]` with dynamic parameter count via `rusqlite::params_from_iter` or build params vector.

2. `pub fn search_by_field(conn: &Connection, field: &str, value: &str, limit: usize) -> Result<Vec<SearchResult>, LibraryError>`:
   - FTS5 field-specific query: `{field}:{value}` syntax
   - Validate field name against allowed set: ["source", "name", "category", "description", "manufacturer", "mpn", "value", "package"]
   - Return LibraryError::NotSupported for unknown fields

3. `pub fn rebuild_index(conn: &Connection) -> Result<(), LibraryError>`:
   - Execute `INSERT INTO components_fts(components_fts) VALUES('rebuild')` to rebuild FTS5 index
   - Useful after bulk operations or if index gets corrupted

4. `pub fn component_count(conn: &Connection) -> Result<usize, LibraryError>`:
   - Simple COUNT(*) on components table

Update `lib.rs` to add `pub mod search;`.

IMPORTANT: FTS5 bm25() returns NEGATIVE scores. More negative = better match. ORDER BY rank (ascending) gives best matches first.
