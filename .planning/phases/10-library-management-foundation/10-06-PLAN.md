Create `preview.rs` with:

`FootprintPreview` struct (Debug, Clone, Serialize, Deserialize):
- `name: String`
- `pads: Vec<PadInfo>`
- `outlines: Vec<OutlineSegment>`
- `courtyard: Option<BoundingBox>`
- `description: Option<String>`

Supporting structs:
- `PadInfo`: name (String), x (f64), y (f64), width (f64), height (f64), shape (PadShape enum: Rect, Circle, RoundRect, Oval)
- `OutlineSegment`: start_x, start_y, end_x, end_y, layer (String)
- `BoundingBox`: min_x, min_y, max_x, max_y (all f64)

`pub fn extract_preview(footprint_data: &str) -> Result<FootprintPreview, LibraryError>`:
- Parse the stored S-expression (footprint_data field from Component)
- Extract pad positions and sizes from `(pad ...)` elements
- Extract outline from `(fp_line ...)` elements on F.Fab or F.SilkS layers
- Extract courtyard from `(fp_line ...)` on F.CrtYd layer, compute bounding box
- Extract name from `(footprint "NAME" ...)`
- Return FootprintPreview with extracted geometry

This is a READ-ONLY extraction - it does not render. The viewer/frontend will use this data to draw the preview. Keep it simple: extract coordinates, let the renderer handle drawing.

Add to `manager.rs`:
- `pub fn get_footprint_preview(&self, source: &str, name: &str) -> Result<Option<FootprintPreview>, LibraryError>`:
  - Get component, check footprint_data is Some
  - Call extract_preview on the footprint_data
  - Return the preview

Update `lib.rs` to add `pub mod preview;`.
  </action>
