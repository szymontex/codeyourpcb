  <files>

  <action>
1. **src/fs.rs** - Define the platform-agnostic trait:
   ```rust
   use crate::error::PlatformError;
   use async_trait::async_trait;

   pub trait FileHandle: Send + Sync {
       fn name(&self) -> &str;
       // No path() method - not available in WASM
   }

   #[async_trait(?Send)]  // ?Send for WASM compatibility
   pub trait FileSystem {
       type Handle: FileHandle;

       async fn pick_file(&self, filters: &[(&str, &[&str])]) -> Result<Option<Self::Handle>, PlatformError>;
       async fn pick_save_file(&self, default_name: &str, filters: &[(&str, &[&str])]) -> Result<Option<Self::Handle>, PlatformError>;
       async fn read(&self, handle: &Self::Handle) -> Result<Vec<u8>, PlatformError>;
       async fn read_string(&self, handle: &Self::Handle) -> Result<String, PlatformError>;
       async fn write(&self, handle: &Self::Handle, data: &[u8]) -> Result<(), PlatformError>;
   }
   ```

2. **src/fs_native.rs** - Native implementation:
   - `NativeHandle` struct with `path: std::path::PathBuf` and `name: String`
   - Implement `FileHandle` for `NativeHandle`
   - Add `NativeHandle::path(&self) -> &std::path::Path` method (native-only, not on trait)
   - `NativeFileSystem` struct (unit struct)
   - Implement `FileSystem` for `NativeFileSystem`:
     - `pick_file`: Use `rfd::AsyncFileDialog::new()`, add filters, call `.pick_file().await`, map to NativeHandle
     - `pick_save_file`: Use `rfd::AsyncFileDialog::new().set_file_name(default_name)`, add filters, call `.save_file().await`
     - `read`: Use `tokio::fs::read(&handle.path).await`
     - `read_string`: Use `tokio::fs::read_to_string(&handle.path).await`
     - `write`: Use `tokio::fs::write(&handle.path, data).await`

3. **src/fs_web.rs** - Web/WASM implementation:
   - `WebHandle` struct with `name: String` and `file: web_sys::File` (for read) or handle data
   - Implement `FileHandle` for `WebHandle` (note: no Send+Sync in WASM, use `?Send` on async_trait)
   - `WebFileSystem` struct (unit struct)
   - Implement `FileSystem` for `WebFileSystem`:
     - `pick_file`: Use `rfd::AsyncFileDialog::new()` (rfd supports WASM too), map to WebHandle. rfd handles the input element fallback internally.
     - `pick_save_file`: Use `rfd::AsyncFileDialog::new().save_file().await` for WASM
     - `read`: Get file bytes from rfd FileHandle's `.read().await`
     - `read_string`: Convert bytes to String
     - `write`: Use rfd FileHandle's `.write()` if available, otherwise create Blob and trigger download

   **Important notes:**
   - Use `#[async_trait(?Send)]` on WASM impls (single-threaded)
   - Use `#[async_trait]` on native impls (multi-threaded)
   - The `rfd` crate works on WASM too - leverage it rather than raw web-sys for file picking
   - For WebHandle, store the rfd::FileHandle internally to allow read operations
   - Web write/save may need to fall back to download trigger if File System Access API unavailable
