Create snapshot.rs with serializable types for JS consumption:

```rust
use serde::Serialize;

#[derive(Serialize)]
pub struct BoardSnapshot {
    pub board: Option<BoardInfo>,
    pub components: Vec<ComponentInfo>,
    pub nets: Vec<NetInfo>,
}

#[derive(Serialize)]
pub struct BoardInfo {
    pub name: String,
    pub width_nm: i64,
    pub height_nm: i64,
    pub layer_count: u8,
}

#[derive(Serialize)]
pub struct ComponentInfo {
    pub refdes: String,
    pub value: String,
    pub x_nm: i64,
    pub y_nm: i64,
    pub rotation_mdeg: i32,
    pub footprint: String,
    pub pads: Vec<PadInfo>,
}

#[derive(Serialize)]
pub struct PadInfo {
    pub number: String,
    pub x_nm: i64,           // relative to component
    pub y_nm: i64,
    pub width_nm: i64,
    pub height_nm: i64,
    pub shape: String,       // "circle", "rect", "roundrect", "oblong"
    pub layer_mask: u32,
    pub drill_nm: Option<i64>,
}

#[derive(Serialize)]
pub struct NetInfo {
    pub name: String,
    pub id: u32,
    pub connections: Vec<PinRef>,
}

#[derive(Serialize)]
pub struct PinRef {
    pub component: String,
    pub pin: String,
}
```

All types use i64/i32/u32 primitives that serialize cleanly to JS numbers.
Shape is serialized as string for simplicity in JS.

Implement PcbEngine with wasm-bindgen exports:

```rust
use wasm_bindgen::prelude::*;
use cypcb_parser::parse;
use cypcb_world::{BoardWorld, sync_ast_to_world};
use cypcb_world::footprint::FootprintLibrary;

mod snapshot;
use snapshot::*;

#[wasm_bindgen]
pub struct PcbEngine {
    world: BoardWorld,
    footprint_lib: FootprintLibrary,
    source: String,
}

#[wasm_bindgen]
impl PcbEngine {
    #[wasm_bindgen(constructor)]
    pub fn new() -> PcbEngine { ... }

    /// Parse source, update world, return error string (empty if success)
    pub fn load_source(&mut self, source: &str) -> String { ... }

    /// Get board snapshot for rendering
    pub fn get_snapshot(&self) -> JsValue { ... }

    /// Query components at a point (returns JSON array of refdes strings)
    pub fn query_point(&self, x_nm: i64, y_nm: i64) -> JsValue { ... }
}
```

Implementation details:
1. `new()`: Initialize empty BoardWorld and FootprintLibrary
2. `load_source()`: Call parse(), then sync_ast_to_world(), collect errors as string
3. `get_snapshot()`: Build BoardSnapshot from ECS world, serialize via serde_wasm_bindgen::to_value
4. `query_point()`: Use spatial index, return array of refdes strings

The `get_snapshot()` method must iterate ECS entities and populate snapshot structs.
Use world.components() to iterate, get footprint from FootprintLibrary for pad info.
