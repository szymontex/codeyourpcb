<objective>
Add LSP-like features (diagnostics, auto-completion, hover) to the Monaco editor using the existing WASM engine as the source of truth, without requiring a separate LSP server connection for web mode.

Purpose: EDIT-02 (auto-completion), EDIT-03 (inline errors), and EDIT-09 (LSP connection). Rather than connecting to the tower-lsp server over WebSocket (which would require a backend server), we bridge the existing WASM engine's diagnostics directly to Monaco's marker and provider APIs. The WASM engine already parses .cypcb and reports errors/violations - we just need to translate those to Monaco's format. For desktop (Tauri), the same approach works since the WASM engine runs in-browser regardless.

Output: Inline error highlighting, keyword auto-completion, and hover documentation in the Monaco editor.
</objective>

**Create `viewer/src/editor/lsp-bridge.ts`:**

This module bridges WASM engine output to Monaco editor features without a real LSP server.

**1. Diagnostics (EDIT-03):**

Export a `updateDiagnostics(monaco, editor, parseErrors, violations)` function:
- `parseErrors`: the string returned by `engine.load_source()` (newline-separated error messages)
- `violations`: the `ViolationInfo[]` from `snapshot.violations`
- Parse error strings to extract line numbers if possible. The engine returns errors like `"Line 5: unexpected token 'foo'"`. Use regex `/[Ll]ine\s+(\d+)/` to extract line numbers. If no line number, mark line 1.
- Convert each error to a Monaco marker:
  ```typescript
  {
    severity: monaco.MarkerSeverity.Error,
    message: errorMessage,
    startLineNumber: lineNum,
    startColumn: 1,
    endLineNumber: lineNum,
    endColumn: model.getLineMaxColumn(lineNum),
  }
  ```
- Convert violations to warning markers (use MarkerSeverity.Warning). Violations don't have line numbers (they have x_nm, y_nm positions), so add them as editor-level warnings without specific line placement (line 1, or skip if no reasonable mapping).
- Call `monaco.editor.setModelMarkers(editor.getModel()!, 'cypcb', markers)` to display

**2. Auto-completion (EDIT-02):**

Export a `registerCompletionProvider(monaco)` function:
- Register a completion item provider for language `'cypcb'`
- Provide completions for:
  - **Keywords**: `version`, `board`, `component`, `net`, `footprint`, `trace`, `zone`, `keepout`, plus component types: `resistor`, `capacitor`, `ic`, `connector`, `diode`, `transistor`, `led`, `crystal`, `inductor`, `generic`
  - **Properties**: `size`, `layers`, `value`, `at`, `rotate`, `pin`, `width`, `clearance`, `current`, `from`, `to`, `via`, `layer`, `locked`, `bounds`, `stackup`, `description`, `pad`, `courtyard`
  - **Layer names**: `Top`, `Bottom`, `Inner1`-`Inner4`, `all`
  - **Units**: When cursor is after a number, suggest `mm`, `mil`, `mA`, `A`, `V`, `k`, `M`, `u`, `n`, `p`
- Use `monaco.languages.CompletionItemKind.Keyword` for keywords, `.Property` for properties, `.Enum` for layers, `.Unit` for units
- Each item should have an `insertText` and `documentation` string explaining what it does
- Filter based on the current word prefix using `model.getWordUntilPosition(position)`

**3. Hover (partial EDIT-09):**

Export a `registerHoverProvider(monaco)` function:
- Register a hover provider for language `'cypcb'`
- When hovering over a keyword, show a Markdown tooltip with a brief description:
  - `board` -> "Defines the PCB board dimensions and layer stackup"
  - `component` -> "Places a component on the board (resistor, capacitor, ic, etc.)"
  - `net` -> "Defines an electrical net connecting component pins"
  - `trace` -> "Defines a copper trace routing a net between points"
  - `footprint` -> "Defines a custom component footprint with pads"
  - etc. (cover all keywords with 1-2 sentence descriptions)
- Use `model.getWordAtPosition(position)` to get the hovered word
- Return `{ contents: [{ value: description }] }`

**Provider registration pattern:**
```typescript
export function registerProviders(monaco: typeof import('monaco-editor')): void {
  registerCompletionProvider(monaco);
  registerHoverProvider(monaco);
}
```
