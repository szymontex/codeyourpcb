<objective>
Set up Monaco editor with .cypcb syntax highlighting and split layout alongside the PCB board viewer.

Purpose: This is the foundation for the embedded code editor. Monaco provides EDIT-04 through EDIT-08 for free (line numbers, code folding, find/replace, undo/redo, multi-cursor). The Monarch tokenizer provides EDIT-01 (syntax highlighting). The split layout provides EDIT-10 (side-by-side).

Output: Working Monaco editor panel with .cypcb syntax highlighting, toggleable via Ctrl+E, displayed alongside the existing PCB canvas viewer.
</objective>

DO NOT add any built-in languages - we only use our custom .cypcb language.

<task type="auto">
  <name>Task 2: Create Monarch tokenizer and editor panel with split layout</name>
  <files>viewer/src/editor/cypcb-language.ts, viewer/src/editor/editor-panel.ts, viewer/index.html</files>
  <action>
**Create `viewer/src/editor/cypcb-language.ts`:**

Define the Monarch tokenizer for .cypcb syntax. Use the pattern from 14-RESEARCH.md:
- Keywords: `version`, `board`, `component`, `net`, `footprint`, `trace`, `zone`, `keepout`, `resistor`, `capacitor`, `ic`, `connector`, `diode`, `transistor`, `led`, `crystal`, `inductor`, `generic`
- Properties: `size`, `layers`, `value`, `at`, `rotate`, `pin`, `width`, `clearance`, `current`, `from`, `to`, `via`, `layer`, `locked`, `bounds`, `stackup`, `description`, `pad`, `courtyard`
- Layer names: `Top`, `Bottom`, `Inner1`, `Inner2`, `Inner3`, `Inner4`, `all`
- Token rules:
  - `//` line comments -> `comment`
  - `"..."` strings -> `string`
  - Numbers with units (mm, mil, mA, A, V, k, M, u, n, p, %) -> `number`
  - Plain numbers -> `number`
  - Identifiers matching keywords/properties/layers -> appropriate token
  - Pin references like `R1.1` -> `variable`
  - Delimiters `{}()=,x` -> `delimiter`
- Language configuration: line comments, bracket pairs `{}`, auto-closing for `{}` and `""`, folding markers for `{}`

Export: `cypcbLanguage` (IMonarchLanguage) and `cypcbLanguageConfig` (LanguageConfiguration). Use `import type * as monaco from 'monaco-editor'` for types only to avoid eager loading.

**Create `viewer/src/editor/editor-panel.ts`:**

Export an async `initEditor(container: HTMLElement)` function that:
1. Dynamically imports `monaco-editor` (lazy load)
2. Dynamically imports `applyMonacoTheme` from `../theme/monaco-theme`
3. Dynamically imports `cypcbLanguage` and `cypcbLanguageConfig` from `./cypcb-language`
4. Registers `cypcb` language with `monaco.languages.register({ id: 'cypcb', extensions: ['.cypcb'] })`
5. Sets Monarch tokens provider and language configuration
6. Calls `applyMonacoTheme(monaco)` to wire theme system
7. Creates editor instance with options:
   - `language: 'cypcb'`
   - `automaticLayout: true`
   - `minimap: { enabled: false }` (save space for side-by-side)
   - `fontSize: 14`
   - `lineNumbers: 'on'` (EDIT-04)
   - `folding: true` (EDIT-05)
   - `wordWrap: 'off'`
   - `scrollBeyondLastLine: false`
8. Returns the editor instance

Also export a `toggleEditorPanel()` function and `isEditorVisible()` function that show/hide the editor container by setting `display: none` / `display: block` on the editor-container element and adjusting canvas-container to fill. Store visibility state.

**Update `viewer/index.html`:**

```html
<div id="main-content" style="display: flex; flex: 1; overflow: hidden;">

  </div>
</div>
```

The editor starts hidden (`display: none`). The existing canvas-container content (canvas, drop-hint, status, routing-status, error-panel) stays unchanged inside canvas-container.

Add a toggle button in the toolbar (before the Open button):
```html
<button id="editor-toggle" title="Toggle Editor (Ctrl+E)">Editor</button>
```

Style the editor-toggle button like other toolbar buttons.

Add CSS for the divider drag handle:
```css

```

IMPORTANT: The `#canvas-container` selector in existing CSS uses `flex: 1` which will work correctly inside the new flex parent. Verify no CSS breaks.
