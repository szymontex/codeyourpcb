<objective>
Wire the Monaco editor into the application lifecycle: file open populates editor, editor changes update the board viewer in real-time, and the divider is draggable for layout customization.

Purpose: The editor and viewer must stay synchronized. The editor becomes the authoritative source of content - typing in the editor feeds the WASM engine for live preview. File operations (open, save, hot reload, desktop events) update the editor which cascades to the viewer.

Output: Fully integrated editor-viewer experience where editing .cypcb code produces immediate visual feedback in the board viewer.
</objective>

  <files>viewer/src/main.ts, viewer/src/editor/editor-panel.ts</files>
  <action>
**Update `viewer/src/main.ts` init() function:**

1. After WASM engine loads and before the render loop, lazily initialize the editor:
   ```typescript
   // Import editor module
   const { initEditor, toggleEditorPanel, isEditorVisible, setEditorContent, getEditorInstance } = await import('./editor/editor-panel');

   // Initialize editor (lazy - Monaco loads on first toggle)
   const editorContainer = document.getElementById('editor-container')!;
   let editorReady = false;
   let editorInstance: any = null; // monaco.editor.IStandaloneCodeEditor
   ```

2. Wire the editor-toggle button:
   ```typescript
   const editorToggleBtn = document.getElementById('editor-toggle')!;
   editorToggleBtn.addEventListener('click', async () => {
     if (!editorReady) {
       editorInstance = await initEditor(editorContainer);
       editorReady = true;
       // Set initial content if a file is loaded
       if (lastLoadedSource) {
         editorInstance.setValue(lastLoadedSource);
       }
       setupEditorSync(editorInstance);
     }
     toggleEditorPanel();
     // Trigger canvas resize
     resize();
   });
   ```

3. Add Ctrl+E keyboard shortcut in the existing keydown handler:
   ```typescript
   if (e.ctrlKey && e.key === 'e') {
     e.preventDefault();
     editorToggleBtn.click();
   }
   ```

4. Create `setupEditorSync(editor)` function that:
   - Listens to `editor.onDidChangeModelContent` with 300ms debounce
   - On change: gets `editor.getValue()`, calls `engine.load_source(content)`, updates `snapshot = engine.get_snapshot()`, sets `dirty = true`, updates `lastLoadedSource = content`, updates error badge
   - Stores a flag `editorIsSource = true` to prevent circular updates

5. Update ALL file loading paths to also set editor content:
   - In `handleFileLoad()`: after `engine.load_source(content)`, add `if (editorReady) editorInstance.setValue(content);`
   - In `reload()` (hot reload): after `engine.load_source(content)`, add `if (editorReady) editorInstance.setValue(content);`
   - In the `openBtn` click handler (web File System Access): add same pattern
   - In the `desktop:open-file` event handler: add same pattern
   - In the `desktop:new-file` event handler: add `if (editorReady) editorInstance.setValue('');`

6. Update save operations to use editor content as authoritative source:
   - In `handleSaveFile()`: if `editorReady`, use `editorInstance.getValue()` instead of `lastLoadedSource`
   - In `desktop:content-request` handler: if `editorReady`, respond with `editorInstance.getValue()`

IMPORTANT: When setting editor content programmatically (file open, hot reload), temporarily disable the sync listener to prevent the editor change from re-triggering `load_source()`. Use a flag like `suppressSync = true` before `setValue()`, and check it in the debounced handler.

**Update `viewer/src/editor/editor-panel.ts`:**

Add a `getEditorInstance()` export that returns the current editor instance (or null if not initialized).
