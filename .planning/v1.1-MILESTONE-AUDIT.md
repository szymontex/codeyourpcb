### Integration Gaps: 3 (all tech debt, no critical blockers)

**Gap 1: File Association Launch Broken**
- `src-tauri/src/lib.rs:27` emits `app.emit("file-opened", { path, content })` when CLI arg is a `.cypcb` file
- `viewer/src/desktop.ts` has no `tauriEvent.listen('file-opened', ...)` call
- **Effect:** Double-clicking `.cypcb` in OS file manager opens app but doesn't load the file
- **Fix:** Add `listen('file-opened', handler)` in `initDesktop()` that dispatches `desktop:open-file`
- **DESK requirement affected:** DESK-01 (partially — File > Open menu works, file-association doesn't)

**Gap 2: Desktop Save Has No Status Feedback**
- `desktop.ts:167,198` dispatches `desktop:file-saved` event after successful IPC save
- `main.ts` has no `addEventListener('desktop:file-saved', ...)` listener
- **Effect:** After File > Save on desktop, status bar shows no "Saved" confirmation (web save correctly shows "Saved" at `main.ts:951`)
- **Fix:** Add `desktop:file-saved` listener in main.ts updating status text

**Gap 3: Edit Menu Items Are Silent No-Ops**
- `src-tauri/src/menu.rs:18-25` defines Undo/Redo/Cut/Copy/Paste menu items
- `handleMenuAction()` in `desktop.ts:72-104` has no cases for `edit.*` actions
- All fall through to `console.log('[Desktop] Unhandled menu action:', action)`
- **Effect:** Misleading UX — items appear enabled but do nothing
- **Root cause:** Expected — no undo system in v1.1. Menu items should be disabled or removed.
- **Fix:** Either remove edit menu items, disable them, or hide until undo system exists
